## 목표 (Goal)

검증 통과한 제보·사건을 일반 대중(중도·반대층·콘텐츠 제작자·연구자)이 탐색하는 **공개 아카이브 화면**(Vite+React)을 정의한다: 리스트, 검색, 카테고리/지역 필터, 상세. **검증 통과분만 공개**하며 미검증은 절대 노출하지 않는다. 0002 공개 조회 API와 0004 판정 결과 위에 올라간다.

## 비목표 (Non-goals)

- 지도 시각화·관계 그래프·타임라인 인터랙션 — 2차(비목표).
- AI 요약/추천, 게임화 — 2차(비목표).
- 댓글·공감·신고 등 사용자 상호작용 — MVP 비목표.
- 로그인/개인화 — 공개 아카이브는 비로그인 열람.
- 데이터 내보내기(CSV/API 공개) — 후속(연구자 대상이지만 MVP는 열람만).

## 사용자 흐름 (User flow)

1. **리스트**: `GET /api/reports`(0002, verified만) → 카드/행 목록. 페이지네이션 또는 무한 스크롤.
2. **검색**: 키워드 입력 → 제목/본문 텍스트 검색(verified 범위 내).
3. **필터**: 카테고리·지역(시도/시군구)·선거(election) 필터. 조합 적용 시 결과·개수 갱신.
4. **상세**: 항목 클릭 → `GET /api/reports/:id`(verified만) → 본문·첨부·출처(0001 스냅샷)·판정 요약(0004의 confidence/validity/severity 등) 표시. 검증 정보·근거 링크 노출로 객관성 전달.
5. 미검증/없는 ID 직접 접근 → **빈/404 화면**(존재 누설 없음).

## 수용 기준 (Acceptance criteria — 테스트 가능하게)

- [ ] 리스트는 **verified=true 인 항목만** 렌더한다(미검증 항목은 결과에 없음 — 모킹 데이터로 단언).
- [ ] 미검증/없는 report ID로 상세 URL 직접 접근 시 **빈/Not Found** 화면을 보이고 본문을 노출하지 않는다.
- [ ] 키워드 검색이 제목/본문에 매칭되는 verified 항목만 반환한다.
- [ ] 카테고리·지역·선거 필터 적용 시 결과가 좁혀지고 결과 개수가 갱신된다. 필터 조합이 동작한다.
- [ ] 상세에 출처(스냅샷/아카이브 링크)와 판정 요약(검증 방법·근거 링크 포함)이 표시된다.
- [ ] 결과 없음/로딩/에러 상태가 각각 명확히 표시된다.
- [ ] 첨부 이미지의 위치정보(exif GPS) 등 민감정보가 공개 화면에서 노출되지 않는다(결정 4).
- [ ] 페이지 어디에도 "부정선거 단정" 단정 카피가 아닌, 데이터/검증 중심 톤이 유지된다(고정 카피 단언).

## 설계 메모 (Design notes)

### 결정 / 근거 (자율 진행)

1. **공개 노출 = API 권위에 위임**. 0002가 verified만 반환하므로 프런트는 그 응답만 신뢰(클라이언트 필터로 미검증을 거르지 않음 — 애초에 안 받음). 미검증 누설 위험을 서버 단일 지점에 둠.
2. **검색**: MVP는 0002 API의 `q` 파라미터(제목/본문 ILIKE 또는 기본 텍스트검색). 전용 검색엔진(OpenSearch 등)은 후속. 본 스펙은 그 API 파라미터를 전제(0002에 `q`·filter 파라미터 추가 필요 — 아래 의존 메모).
3. **판정 표시**: confidence/validity/severity/legal_issue + 검증 방법·근거 링크를 객관적으로 표기(점수/배지). "단정" 아닌 "검증 결과" 프레이밍.
4. **민감정보**: 공개 첨부는 exif GPS 등 제거된 버전 노출 또는 메타 비표시. 원본 메타는 내부(0004)만. (실제 stripping 책임 위치는 구현 시 0002/이미지 파이프와 1줄 합의 — 기본: 공개 직렬화에서 exif 위치 제외.)
5. **출처 신뢰성**: 출처는 0001 스냅샷(captured_at·content_hash·archive_url) 기반으로, 원본이 바뀌어도 "우리가 본 시점"을 함께 표기.

### 0002 의존 메모 (착수 전 확인)
- 0005는 0002 `GET /api/reports`에 **검색(q)·필터(category/region/election)·페이지네이션** 파라미터가 필요. 0002 구현 시 포함하거나, 없으면 0002에 소규모 추가 스펙으로 반영. (0002 본문은 목록/상세를 이미 정의 — 파라미터 확장만.)
