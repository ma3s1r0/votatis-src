## 목표 (Goal)

기능은 갖춰졌으나 비주얼이 "민짜"(기본 폼·여백 부족)다. 0010 토큰 위에 **응집된 시각 디자인**을 입혀, 신뢰 가는 시빅 데이터 서비스처럼 보이게 한다. 라이브 스크린샷으로 반복 검증.

## 비목표 (Non-goals)

- 기능/문구/마크업 의미 변경(스타일만). DOM 구조·role·text·data-testid 보존 → 기존 테스트 회귀 0.
- 픽셀 단위 Figma 정합(편집권한 부재). 다크테마.
- 신규 기능.

## 사용자 흐름

전 페이지(홈·제보 마법사·공개 아카이브 목록/상세·관리 로그인/큐/판정)의 시각 품질을 끌어올린다.

## 수용 기준 (테스트 가능 + 스크린샷 검증)

- [ ] 인라인 raw 스타일을 줄이고 **공용 스타일시트/클래스**(토큰 기반)로 정돈. 토큰 단일 출처 유지.
- [ ] **버튼**: 주(accent 채움)·보조(아웃라인) 스타일 + hover/focus 링. 기본 브라우저 버튼 잔존 최소화.
- [ ] **입력/셀렉트/textarea**: 일관 패딩·보더·radius·focus 링.
- [ ] **헤더**: 정돈된 네비(로고 강조, 링크 간격, 현재 위치 표시), 구분선/배경.
- [ ] **홈**: 히어로(헤드라인+서브+CTA 2개) + 간단한 원칙/설명 영역. 스캐폴드 잔재 없음.
- [ ] **아카이브**: 목록을 카드/정돈된 행 + hover, 필터바 카드, 페이지네이션 정돈. 상세는 섹션 카드 + 검토요약 강조.
- [ ] **배지**: 심각도(1~5)·유효성·verified를 색 배지/pill로(의미색: 심각도 높을수록 강조, 빨강은 위험만).
- [ ] **마법사**: 단계 진행 표시 개선(스텝 인디케이터), 폼·요약·완료 화면 정돈.
- [ ] **관리 콘솔**: 큐·판정 폼·로그인 동일 디자인 언어 적용.
- [ ] 반응형: 좁은 폭에서 깨지지 않음(필터 wrap·컨테이너 패딩).
- [ ] 게이트 `pnpm -r typecheck && build && test` 통과, 기존 web/api 테스트 회귀 0.

## 테스트 계획

- 스타일은 RTL로 픽셀 검증 불가 → **DOM/testid 보존**으로 기존 테스트 회귀 0이 1차 게이트.
- 최소 토큰/클래스 적용 단언(예: 버튼이 공용 className 보유) 몇 건.
- **시각 검증 = 헤드리스 브라우저 스크린샷 리뷰**(오케스트레이터가 반복 확인·피드백).

## 설계 메모

- 톤: 중립·신뢰·데이터 서비스. accent=teal 유지, 빨강은 위험/에러만. 충분한 여백·명확한 위계.
- 토큰 부족분(예: 추가 음영·라운드·muted 배경)은 `tokens.css`에 보강.
