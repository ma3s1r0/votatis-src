## 목표 (Goal)
현재 5단계 위저드(`ReportWizard`)를 Figma 정본 02(제보폼)에 맞춰 **단일 스크롤 페이지**로 전환한다. 한 화면에서 도메인 선택 → 사진 첨부 → 위치 → 분류 → 상세 설명 → 동의 → 제출까지 끊김 없이 진행한다. 제출·EXIF 검증·첨부 업로드·접수번호 추적·완료 화면 등 **기존 로직과 데이터 계약은 그대로 보존**한다.

## 비목표 (Non-goals)
- 백엔드 계약(`createReport`/`uploadAttachment`) 변경 없음.
- EXIF 1차 검증/차단·경고 규칙(0015) 변경 없음.
- 지역 구조화 데이터(시도/시군구/읍면동 cascade) 제거 없음 — Figma의 단일 위치 입력 대신 구조화 select를 유지(데이터 무결성).
- 접수번호 추적·"내 제보" 저장(0013) 동작 변경 없음.

## 사용자 흐름 (User flow)
`/report` 단일 페이지. 위에서 아래로:
1. 헤더: ← 뒤로 + "{도메인} 증거 제보" 제목 + 검수/책임 안내 문구.
2. 도메인 세그먼트(선거 의혹 / 집회 현장) — 전환 시 분류 초기화.
3. 사진/PDF 첨부(드롭존) — EXIF 차단 시 인라인 차단 카드, 경고 시 경고 문구.
4. 위치: 시도 → 시군구 → 읍면동 cascade.
5. 의혹 유형(분류) + 선거(선택).
6. 제목(필수) + 상세 설명(본문) + 발생 시점.
7. 동의 체크박스 + "제보 제출" 버튼(동의 전 비활성).
제출 성공 → 완료 화면(접수번호·복사·상태조회/홈/새 제보).

## 수용 기준 (Acceptance criteria — 테스트 가능하게)
- [ ] 단계 인디케이터(stepper)·"다음/이전" 버튼·"N/M 단계" 표기가 없다.
- [ ] 진입 즉시 모든 입력(도메인·첨부·위치·분류·제목·본문·동의·제출)이 한 화면에 보인다.
- [ ] 도메인을 집회 현장으로 바꾸면 분류 옵션이 assembly 분류로 바뀐다.
- [ ] 시도 선택 전 시군구는 비활성, 시도 선택 후 해당 시군구 목록이 채워진다.
- [ ] 제목 미입력으로 제출하면 제출되지 않는다(필수). 동의 미체크 시 제출 버튼 비활성.
- [ ] 비원본(EXIF 없음) 첨부는 차단되고 차단 사유가 표시된다(0015).
- [ ] 정상 제출 시 createReport 호출 → (첨부 있으면) uploadAttachment → 완료 화면에 접수번호 표시 + "내 제보" 저장.
- [ ] 완료 화면에서 접수번호 복사·상태조회 링크가 동작한다.
- [ ] 공통 토큰 클래스(.container/.btn/.field 등) 사용(인라인 style 제거).

## 테스트 계획 (TDD — Red 먼저)
기존 `ReportWizard.*.test.tsx`를 단일 페이지 기준으로 갱신한다(스텝 네비 클릭 제거, 동작 단언 유지):
- nav/progress: 스텝 개념 제거 → "단일 페이지에 전체 입력 노출 + 제출 게이트"로 재작성.
- domain/category/region/attach/exif(Block/Warn)/submit/summary/tracking/clipboard/copy/done/qa/tokens: 스텝 이동 제거 후 동작 단언 유지.

## 설계 메모 (Design notes)
- Figma 02엔 제목 필드가 없으나 백엔드는 title 필수 → 제목 입력 유지(상세 설명 위).
- 컴포넌트 인라인 style → `app.css` 공용 클래스(.form-grid/.field/.input/.section-card 등)로 이관.
- 초안 자동저장(sessionStorage)·완료 화면·startNewReport 로직 보존(step 필드만 제거).
