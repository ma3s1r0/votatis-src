## 목표 (Goal)

시민이 제보를 쉽게 올리도록 **단계별 마법사 UI**(Vite+React)를 정의한다: 상황 설명 → 분류 → 지역 선택 → 출처·사진 첨부 → 제출. 0002 API(생성 + 2단계 첨부 presigned)를 연동한다. 톤은 객관적 데이터 서비스 — "부정선거 단정"이 아니라 검증 가능한 정황 기록을 유도하는 문구.

## 비목표 (Non-goals)

- 제보 생성/첨부/조회 **API** (→ 0002). 본 스펙은 프런트 UI·연동만.
- 라벨링/검증 (→ 0004), 공개 출력 (→ 0005).
- 시민 계정/로그인 — 제보는 익명(0002). 로그인 화면 없음.
- 지도 기반 지역 선택, 자동 지오코딩 — 2차. MVP는 드롭다운(시도→시군구→읍면동) 텍스트 선택.
- 임시저장(서버 draft)·다중 기기 이어쓰기 — MVP는 클라이언트 로컬 상태만.

## 사용자 흐름 (User flow)

1. **Step 1 상황 설명**: 제목 + 본문(무엇을 보았는지) + 발생 시점. 단정 대신 관찰 기록을 유도하는 안내 문구.
2. **Step 2 분류**: 사건 카테고리 선택(0001 category). 어떤 선거인지(election) 선택/연결.
3. **Step 3 지역 선택**: 시도→시군구→읍면동 단계 드롭다운(0001 region 컬럼).
4. **Step 4 출처·사진**: 출처 URL/설명(0001 source) + 사진/PDF 첨부. 첨부는 0002 create→S3 PUT→finalize.
5. **Step 5 제출 확인**: 입력 요약 + 동의(consent/license, 0001) 체크 → `POST /api/reports` → 완료 화면(접수번호/안내). "검증 후 공개될 수 있음" 고지.

각 단계: 이전/다음 이동, 미충족 시 다음 비활성, 진행 표시. 새로고침 시 로컬 상태 복원(결정 4).

## 수용 기준 (Acceptance criteria — 테스트 가능하게)

- [ ] 마법사가 5단계로 렌더되고, 현재 단계/진행도가 표시된다.
- [ ] 필수 입력 미충족 시 "다음" 버튼이 비활성(또는 진행 차단)되고, 해당 필드 에러가 표시된다.
- [ ] 지역 선택은 시도 선택 후에만 시군구가, 시군구 선택 후에만 읍면동이 활성화된다.
- [ ] 첨부 추가 시 0002 흐름대로 create→PUT→finalize 가 순서대로 호출된다(모킹). 업로드 진행/완료 상태가 표시된다.
- [ ] 허용 외 파일(미허용 mime/초과 size)은 클라이언트에서 미리 거부하고 안내 메시지를 보인다(서버 거부에도 대응).
- [ ] consent 미동의 시 제출 버튼이 비활성.
- [ ] 제출 성공 시 완료 화면과 접수 안내가 보이고, 마법사 상태가 초기화된다.
- [ ] 제출 실패(API 400/429 등) 시 사용자에게 명확한 에러가 표시되고 입력이 보존된다.
- [ ] 안내 문구가 "단정"이 아닌 관찰·기록 톤(고정 카피 존재 단언).

## 설계 메모 (Design notes)

### 결정 / 근거 (자율 진행)

1. **상태 = 클라이언트 로컬**(마법사 단계 상태 + 새로고침 복원은 sessionStorage). 서버 draft 없음 — MVP 단순화, 익명 제보라 서버 보관 동기 약함.
2. **첨부 = 0002 2단계 그대로**. UI는 finalize 성공까지를 "첨부 완료"로 표시. 제출(POST reports)은 본문 먼저 또는 첨부 먼저든 0002 계약을 따름(report 생성 후 attachment create가 자연스러움 → report_id 필요).
   - 순서: Step5 제출 시 report 생성 → 직전 단계에서 고른 파일들을 그 report_id로 create/finalize. (또는 report를 Step1에서 선생성하는 대안은 미채택 — 미완성 제보 누적 방지.)
3. **지역 데이터**: 시도/시군구/읍면동 목록은 정적 데이터셋(번들 또는 정적 JSON). 외부 API 의존 없음(MVP).
4. **새로고침 복원**: 진행 중 입력은 sessionStorage 저장, 제출 성공 시 클리어.
5. **접근성/톤**: 라벨·에러 명시, 단정 표현 배제 카피. 디자인 정본은 [[votatis-figma]] 참조.

### 무결성
- 발생 시점은 사용자가 입력하되, `collected_at`(수집 시각)은 0002 서버가 기록 — UI는 관여 안 함.
