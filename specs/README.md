# Specs — 스펙 인덱스 (SDD)

모든 기능은 **스펙 먼저**(Spec-Driven Development). 코드를 짜기 전에 여기 스펙 문서를 만들고, 그 수용 기준을 **테스트로 먼저 작성**(TDD)한 뒤 구현한다.

## 디렉터리 (상태 = 위치)

```
specs/
  _template.md     # 새 스펙은 이걸 복사해서 시작
  not-started/     # 작성됨, 구현 전
  in-progress/     # 구현 중 (테스트 먼저 → 통과)
  in-review/       # 구현·검증 완료, 리뷰 대기
  completed/       # 리뷰 통과
```

상태 전환은 **파일 이동**으로 표현한다. 파일명: `NNNN-kebab-title.md` (NNNN = 0001부터).

## 워크플로 (SDD + TDD, 필수)

1. **Spec**: `_template.md` 복사 → `not-started/NNNN-*.md`. 목표·비목표·사용자 흐름·**수용 기준(테스트 가능한 형태)**·테스트 계획 작성. 모호하면 가정 나열 후 질문(추측 금지).
2. **Red**: 구현 전, 수용 기준을 **실패하는 테스트**로 작성. 스펙을 `in-progress/`로 이동.
3. **Green**: 테스트를 통과시키는 **최소 구현**. 투기적 추가 금지.
4. **Refactor**: 테스트 그린 유지하며 정리.
5. **Review**: `pnpm -r typecheck && pnpm -r build && pnpm -r test` 통과 → `in-review/`로 이동, QA/리뷰 인계. 통과하면 `completed/`.

## 권장 개발 순서 (MVP 백로그)

```
0006 인증 ─┐
           ├─→ 0004 라벨링 콘솔 ─→ 0005 공개 아카이브
0002 API ──┘                ↑
   └─→ 0003 입력 마법사       │
0005 ← 0002 + 0004 ──────────┘
```

- 1순위 **0006**(인증) — 0004의 선행.
- **0002**(수집 API)는 0006과 **독립이라 병렬 가능**.
- 2순위 **0004**(라벨링) — 0001·0002·0006 의존.
- 3순위 **0003**(입력 마법사) — 0002 의존.
- 마지막 **0005**(공개 아카이브) — 0002·0004 의존.

권장 직렬 순서: **0006 → 0002 → 0004 → 0003 → 0005** (0002는 0006과 병렬 가능).

## 디자인 정본 백로그 (0013~0018, Figma + 서비스 블루프린트 근거)

Figma 디자인 정본과 서비스 블루프린트가 시사하는 **미구현 기능** 묶음. 모든 스펙은 **회귀 0**(기존 0001~0012 테스트 유지)이 절대 조건이며, 인프라가 무거운 부분(실 모자이크·실 지도타일)은 비목표로 명시하고 구조·인터페이스·단위검증까지만 다룬다.

```
0013 접수번호 추적 ── (0001·0002·0003·0004·0005 의존) — 추적 인프라(권장 1순위)
0014 도메인 세그먼트 ── (0001·0002·0003·0004·0005 의존) → 0016 의 분기 키 제공
0015 클라 EXIF 검증 ── (0002·0003 의존, web 전용)
0016 모자이크/원본분리 ── (0001·0002·0004·0008·0014 의존)
0017 2인 교차검증 ── (0001·0004·0006 의존)
0018 지도 뷰 + 조회수 ── (0001·0002·0005 의존)
```

- **0013** 접수번호 추적 — report 에 공개 접수번호(`VT-YYYY-MMDD-NNNN`) 부여 + 인증 없는 상태조회 API(타임라인만)·내 제보(localStorage). 블루프린트 PO★(유저 9명 요청).
- **0014** 도메인 세그먼트 — report.domain(election|assembly, 기본 election) 추가. 제보폼·아카이브·검수 큐 상단 세그먼트. 0016 이 assembly 분기에 의존.
- **0015** 클라 EXIF 검증 — 브라우저에서 EXIF 추출·MIME 이중검증, 비원본이면 "원본 사진이 아닙니다" 차단(화면 11). 서버 무변경, 1차 게이트.
- **0016** 모자이크/원본분리 — 공표 시 assembly 첨부 얼굴 모자이크 공개본 생성, 원본 보존, 공개는 public 경로만. 실 모자이크는 MosaicPort 인터페이스로 격리(비목표 경계).
- **0017** 2인 교차검증 — verified 확정에 서로 다른 reviewer 2인 동의 필요(1/2→2/2). 중복 동의 거부·이력 보존.
- **0018** 지도 뷰 + 조회수 — 시도 중심 좌표 정적 매핑 + SVG 핀(상태별 색·범례 건수), 공개 상세 조회수 +1·표시. 실 지도타일/지오코딩은 비목표.

권장 직렬 순서: **0013 → 0014 → 0015 → 0016 → 0017 → 0018** (0015 는 web 전용이라 0014 와 병렬 가능, 0016 은 0014 선행 필수).
